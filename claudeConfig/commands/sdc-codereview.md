# /sdc-codereview 命令规范

## 审查维度（固定 5 项）

每项按 严重/中等/建议 三级标注：

### 1. 质量（Quality）
- 逻辑是否正确
- 是否有边界情况未处理
- 错误处理是否完善

### 2. 安全（Security）
- 是否有注入风险
- 是否有敏感信息泄露
- 权限控制是否正确
- 对照 rules/security.md 🔴 级别规则逐项检查

### 3. 可维护性（Maintainability）
- 命名是否清晰
- 函数是否过长（建议 ≤ 50 行）
- 是否有重复代码

### 4. 性能（Performance）
- 是否有不必要的循环/查询
- 是否有 N+1 问题
- 缓存使用是否合理

### 5. 测试覆盖（Test Coverage）
- 新增代码是否有测试
- 测试用例是否有效
- 覆盖率是否达标

## 总结
- 严重问题：N 个 → 必须修复
- 中等问题：N 个 → 建议修复
- 建议：N 个 → 可选
